Como usar consulta amanha sem perder tempo

Como o professor liberou pesquisa, codigo antigo e materiais, a estrategia nao e decorar tudo. A estrategia e saber onde procurar e qual padrao copiar com entendimento.

  1. Leia o enunciado e marque os substantivos: produto, livro, tarefa, aluno, item.
  2. Transforme esses substantivos em tabela e colunas.
  3. Monte primeiro o banco.sql.
  4. Teste o SQL no pgAdmin antes de ir para o Node.
  5. No Node, copie o padrao mental: criar cliente, conectar, executar query, tratar erro, fechar conexao.
  6. Faca uma funcao por vez. Nao comece pelo menu.
  7. So monte o menu quando as funcoes principais existirem.

Front-end: HTML, CSS, Bootstrap e DOM

HTML

  • HTML define estrutura: titulos, paragrafos, listas, links, imagens, formularios e tabelas.
  • Use tags semanticas quando fizer sentido: header, main, section, footer.
  • Formulario normalmente usa input, label, button e atributos como type, name, id.

CSS

  • CSS controla aparencia: cores, espacamento, bordas, fontes e layout.
  • Seletores importantes: tag, classe .classe, id #id.
  • Layout mais comum: display: flex, display: grid, gap, padding, margin.
  • Responsividade: use @media, porcentagem, max-width e layouts que quebram em telas menores.

Bootstrap

  • Bootstrap acelera layout com classes prontas.
  • Classes comuns: container, row, col, btn, card, navbar.
  • Se cair Bootstrap, geralmente o foco e montar pagina responsiva sem escrever tanto CSS manual.

DOM

  • DOM e o HTML visto pelo JavaScript como objetos manipulaveis.
  • Buscas comuns: document.querySelector(), document.getElementById().
  • Eventos comuns: click, submit, input.

PostgreSQL: SQL que precisa estar na mao

Fluxo do banco

  1. Criar banco com CREATE DATABASE.
  2. Conectar no banco pelo pgAdmin.
  3. Criar tabela com CREATE TABLE.
  4. Inserir dados iniciais com INSERT INTO.
  5. Testar com SELECT * FROM tabela;.

Modelo base de tabela

CREATE TABLE produtos (
    id SERIAL PRIMARY KEY,
    nome VARCHAR(100) NOT NULL,
    categoria VARCHAR(50),
    preco NUMERIC(10, 2),
    estoque INTEGER,
    descricao TEXT
);

Comandos basicos

Operacao SQL Para que serve
Criar INSERT INTO Adicionar novo registro.
Ler SELECT Buscar dados.
Atualizar UPDATE ... SET ... WHERE Alterar dados existentes.
Remover DELETE FROM ... WHERE Apagar registro.
Nunca faca UPDATE ou DELETE sem WHERE, a menos que voce queira alterar ou apagar tudo.
Node + PostgreSQL: padrao da Loja do Alquimista

Pacotes principais

npm init -y
npm install pg prompt-sync

Conexao

A configuracao do Node usa as mesmas informacoes do pgAdmin: host, porta, usuario, senha e banco.

const { Client } = require('pg');

function criarCliente() {
    return new Client({
        host: 'localhost',
        port: 5432,
        user: 'postgres',
        password: 'sua_senha',
        database: 'nome_do_banco'
    });
}

Padrao de toda funcao com banco

async function minhaFuncao() {
    const client = criarCliente();

    try {
        await client.connect();
        // client.query(...)
    } catch (erro) {
        console.log('Erro:', erro.message);
    } finally {
        await client.end();
    }
}

Por que usar $1, $2, $3?

Porque valores digitados pelo usuario devem entrar como dados, nao como texto colado dentro do SQL. Isso evita erro e reduz risco de SQL injection.

await client.query(
    'SELECT * FROM produtos WHERE nome ILIKE $1',
    [`%${nome}%`]
);

CRUD que voce precisa reconhecer

  • Listar: SELECT * FROM tabela ORDER BY nome.
  • Cadastrar: INSERT INTO tabela (...) VALUES (...) RETURNING *.
  • Atualizar: UPDATE tabela SET campo = $1 WHERE id = $2 RETURNING *.
  • Remover: buscar antes, confirmar, depois DELETE FROM tabela WHERE id = $1.
Servidores HTTP, Express e Full Stack

HTTP puro no Node

O modulo nativo http cria servidor sem instalar nada. Ele trabalha com requisicao e resposta.

  • req: pedido que chegou.
  • res: resposta que sera enviada.
  • listen(3000): deixa o servidor ouvindo em uma porta.

Express

Express organiza servidores e APIs. Ele evita muitos if de rota e facilita trabalhar com caminhos, metodos HTTP e JSON.

const express = require('express');
const app = express();

app.get('/', (req, res) => {
    res.send('Pagina inicial');
});

app.listen(3000);

Full Stack

Full stack e quando front-end e back-end se comunicam. O fluxo basico e:

  1. Usuario preenche formulario no HTML.
  2. JavaScript envia dados para uma rota do back-end.
  3. Back-end valida e salva no banco.
  4. Banco responde.
  5. Back-end devolve resposta para o front-end.

A logica e a mesma que voce ja usa em JavaScript: receber dados, decidir, repetir e organizar em funcoes. Em C, muda principalmente a sintaxe e o controle dos tipos.

  2. Variaveis e tipos: int, float, double e char.
  3. Condicionais: if, else e switch.
  4. Repeticao: while, do while e for.
  5. Funcoes, vetores, strings, matrizes e struct.

Esqueleto minimo em C

#include <stdio.h>

int main() {
    int idade;
    printf("Digite a idade: ");
    scanf("%d", &idade);

    if (idade >= 18) {
        printf("Maior de idade\n");
    }

    return 0;
}
printf mostra dados; scanf recebe dados. O & passa o endereco da variavel para o scanf.
C# do zero e Programacao Orientada a Objetos

C# e diferente de C. Ele roda sobre a plataforma .NET e foi a linguagem escolhida pelo professor para estudar POO. Para a revisao imediata, concentre-se nos capitulos 0 a 4 e em tratamento de erros.

Sintaxe basica

Console.Write("Nome: ");
string nome = Console.ReadLine();

Console.Write("Idade: ");
int idade = int.Parse(Console.ReadLine());

if (idade >= 18) {
    Console.WriteLine($"{nome} e maior de idade");
}
  • Console.WriteLine(): mostra e pula linha.
  • Console.ReadLine(): recebe texto.
  • int.Parse(): converte texto para inteiro.
  • $"...{variavel}...": interpolacao de texto.

Classe, objeto, propriedade e metodo

class Produto
{
    public string Nome { get; set; }
    public decimal Preco { get; set; }

    public Produto(string nome, decimal preco)
    {
        Nome = nome;
        Preco = preco;
    }

    public void Exibir()
    {
        Console.WriteLine($"{Nome}: R$ {Preco}");
    }
}

Produto pocao = new Produto("Pocao", 15.50m);
pocao.Exibir();
  • Classe: molde que define dados e comportamentos.
  • Objeto: instancia criada com new.
  • Construtor: prepara o objeto ao ser criado; tem o mesmo nome da classe.
  • Propriedade: dado controlado por get e set.
  • Metodo: acao que o objeto sabe executar.

Quatro pilares da POO

PilarIdeiaPista em C#

PolimorfismoObjetos diferentes respondem ao mesmo metodo.virtual, override, interfaces.

Modelagem de sistemas e UML

Modelar e criar uma representacao simplificada antes ou junto da implementacao. Serve para visualizar, comunicar, analisar e documentar o sistema.

  • Requisito funcional: o que o sistema deve fazer, como cadastrar produto.
  • Requisito nao funcional: qualidade ou restricao, como seguranca e tempo de resposta.
  • Caso de uso: interacao entre ator e sistema para atingir um objetivo.
  • Diagrama de classes: classes, atributos, metodos e relacionamentos.
  • Diagrama de atividades: fluxo de decisoes e tarefas.
  • Diagrama de sequencia: mensagens trocadas ao longo do tempo.
Testes de sistemas

Teste busca revelar defeitos e verificar se o software atende ao esperado. Qualidade e responsabilidade do processo inteiro, nao apenas de quem testa.

  • Caso de teste: pre-condicao, dados de entrada, passos e resultado esperado.
  • Teste unitario: verifica uma pequena unidade isolada.
  • Teste de integracao: verifica componentes trabalhando juntos, como API e banco.
  • Teste de sistema: avalia o sistema completo.
  • Teste de aceitacao: confirma se atende a necessidade do usuario/cliente.
  • Caixa-preta: testa entradas e saidas sem depender do codigo interno.
  • Caixa-branca: considera caminhos e estrutura interna do codigo.
  • Regressao: verifica se uma mudanca quebrou algo que funcionava.
Exemplo: quantidade valida entre 1 e 100. Teste 1, 100, 0, 101 e um texto. Isso exercita valores validos, limites e entradas invalidas.
Banco de dados: modelagem, SQL e normalizacao
  • Modelo conceitual: entidades e relacionamentos, sem pensar no SGBD.
  • Modelo logico: tabelas, atributos, chaves e relacionamentos.
  • Modelo fisico: SQL implementado no PostgreSQL ou outro SGBD.
  • Chave primaria: identifica cada registro de forma unica.
  • Chave estrangeira: liga uma tabela a outra.
  • Normalizacao: reduz repeticao e inconsistencias.
GrupoComandos comunsUso
DDLCREATE, ALTER, DROPDefine estruturas.
DMLINSERT, UPDATE, DELETEManipula registros.
DQLSELECTConsulta dados.
DCLGRANT, REVOKEControla permissoes.

Para combinar tabelas, revise INNER JOIN, LEFT JOIN, a condicao ON e a relacao entre chave primaria e estrangeira.

Erros comuns e como diagnosticar rapido
Erro Causa provavel O que conferir
Senha invalida Senha do PostgreSQL diferente no codigo. password em criarCliente().
Banco nao existe Nome do banco errado ou nao criado. database e pgAdmin.
Tabela nao existe Voce criou banco, mas nao executou CREATE TABLE nele. Se esta conectado no banco certo.
Programa trava no menu Faltou await em funcao async ou loop mal controlado. Chamadas como await listarItens().
Erro de sintaxe Chave, parenteses, crase ou virgula faltando. Use node --check app.js.
UPDATE nao altera nada ID nao existe ou parametros invertidos. WHERE id = $2 e array de valores.
Roteiro para montar qualquer sistema do simulado

Passo a passo recomendado

  1. Defina o tema: loja, biblioteca, tarefas, mercado, farmacia.
  2. Defina a tabela principal e colunas.
  3. Crie banco.sql e teste no pgAdmin.
  4. Crie projeto Node com npm init -y.
  5. Instale pg e prompt-sync.
  6. Crie criarCliente().
  7. Implemente listar e teste.
  8. Implemente cadastrar e teste.
  9. Implemente atualizar e teste.
  10. Implemente remover e teste.
  11. Monte o menu por ultimo.

Checklist final antes de entregar

  • O sistema roda com node app.js ou npm start.
  • Conexao fecha no finally.
  • Valores digitados usam parametros $1, $2.
  • Menu tem opcao de sair.
  • Mensagens de erro sao claras.
  • Voce testou cadastrar, listar, atualizar e remover.
